# KVR OSC Data Platform v1 - ROADMAP

- [ ] Phase 0 – Bestandsaufnahme
  - Ziel: Aktuellen Stand vollständig verstehen
  - Ergebnis: Dokumentierter Ist-Zustand, bekannte Probleme und Datenquellen

- [ ] Phase 1 – Datenbasis stabilisieren
  - Ziel: Single Source of Truth etablieren
  - Ergebnis: osc_master.csv, artists.csv, archive_links.csv, scorecard_index.csv

- [ ] Phase 2 – Importsystem
  - Ziel: Neue Scorecards sicher importieren
  - Ergebnis: Hash-Erkennung, Reimport, Staging, Diff-System

- [ ] Phase 3 – Datenqualität
  - Ziel: Fehler automatisch erkennen
  - Ergebnis: Validator, Issues-System, Qualitätsberichte

- [ ] Phase 4 – Artist Registry
  - Ziel: Zentrale Künstlerdatenbank
  - Ergebnis: Artist IDs, Aliase, Merge-Funktion, Statistiken

- [ ] Phase 5 – Archive.org Integration
  - Ziel: Robuste Player-Zuordnungen
  - Ergebnis: Archive-Crawler, Matching, Review-System

- [ ] Phase 6 – Admin Backend
  - Ziel: Browserbasierte Pflege
  - Ergebnis: Login, Editor, API, Backups, Änderungsprotokoll

- [ ] Phase 7 – Hall of Fame v2
  - Ziel: Visualisierung der Masterdaten
  - Ergebnis: Artists, OSC Browser, Results Explorer

- [ ] Phase 8 – Wartungsbetrieb
  - Ziel: Neue OSCs ohne Codeänderung integrieren
  - Ergebnis: Upload → Review → Publish Workflow

- [ ] Phase 9 – User & Security
  - Ziel: Geschützte Administration
  - Ergebnis: Benutzerverwaltung, Rollenmodell, optional 2FA

- [ ] Phase 10 – Statistik & Analytics
  - Ziel: Nutzung verstehen
  - Ergebnis: Klicktracking, Filtertracking, Analytics Dashboard

- [ ] Phase 11 – Player & Discovery Features
  - Ziel: OSC-Musik erfahrbar machen
  - Ergebnis: Random Track Vorauswahl, Discovery Features

- [ ] Phase 12 – Deployment & Backup
  - Ziel: Langfristig sicherer Betrieb
  - Ergebnis: Backup, Restore, Deployment Dokumentation
